# 非機能要件

## 1. 性能要件

| 要件ID | 要件名 | 目標値 | 測定方法 |
|-------|-------|--------|---------|
| NFR-P-001 | APIレスポンス時間 | P95 ≤200ms | k6負荷テスト |
| NFR-P-002 | 画面表示時間 | 初回ロード ≤3秒 | Lighthouse |
| NFR-P-003 | 同時接続ユーザー数 | 100名以上 | Locust |
| NFR-P-004 | ファイルアップロード | 10MB ≤5秒 | k6 |
| NFR-P-005 | 検索レスポンス | ≤500ms（全文検索含む） | Elasticsearch |
| NFR-P-006 | PDF生成時間 | ≤3秒 | 実測 |
| NFR-P-007 | AI応答時間 | ≤5秒（ストリーミング除く） | 実測 |

---

## 2. 可用性要件

| 要件ID | 要件名 | 目標値 | 備考 |
|-------|-------|--------|------|
| NFR-A-001 | システム稼働率 | ≥99.5%（月次） | 計画メンテナンス除く |
| NFR-A-002 | 計画メンテナンス | 月1回・深夜2〜4時 | 事前告知必須 |
| NFR-A-003 | RTO（目標復旧時間） | P1障害: 2時間以内 | - |
| NFR-A-004 | RPO（目標復旧ポイント） | 1時間以内のデータ損失 | WALアーカイブ活用 |
| NFR-A-005 | 冗長構成 | DBプライマリ/レプリカ構成 | フェイルオーバー対応 |

---

## 3. セキュリティ要件

| 要件ID | 要件名 | 要件内容 |
|-------|-------|---------|
| NFR-S-001 | 通信暗号化 | 全通信をHTTPS（TLS1.2以上）で暗号化 |
| NFR-S-002 | データ暗号化 | 保存データの重要情報（パスワード等）を暗号化 |
| NFR-S-003 | 認証強度 | JWT + MFA（管理者・承認者は必須） |
| NFR-S-004 | RBAC | ロールベースアクセス制御の全エンドポイント適用 |
| NFR-S-005 | 監査ログ | 全操作の監査ログ記録（最低5年間保管） |
| NFR-S-006 | 脆弱性対策 | OWASP Top10への対応 |
| NFR-S-007 | SoD | 職務分掌分離の実施 |
| NFR-S-008 | パスワードポリシー | 12文字以上・大小英数記号必須 |
| NFR-S-009 | セッション管理 | アイドルタイムアウト30分 |

---

## 4. 保守性要件

| 要件ID | 要件名 | 要件内容 |
|-------|-------|---------|
| NFR-M-001 | テストカバレッジ | 単体テスト≥80%、統合テスト≥70% |
| NFR-M-002 | コード品質 | Lintエラーゼロ、型チェックエラーゼロ |
| NFR-M-003 | ドキュメント | 全APIのOpenAPI仕様書を常に最新に維持 |
| NFR-M-004 | CI/CD | プッシュ時の自動テスト・デプロイ |
| NFR-M-005 | モジュール独立性 | モジュール間の疎結合設計 |
| NFR-M-006 | ログ | 構造化ログ（JSON形式）での出力 |

---

## 5. 拡張性要件

| 要件ID | 要件名 | 要件内容 |
|-------|-------|---------|
| NFR-E-001 | ユーザー数スケール | 500名まで追加設定なしで対応 |
| NFR-E-002 | データ量スケール | 5年分のデータを格納可能な設計 |
| NFR-E-003 | 水平スケール | アプリケーション層の水平スケールアウト対応 |
| NFR-E-004 | API拡張性 | 新モジュール追加時のAPI拡張が容易な設計 |
