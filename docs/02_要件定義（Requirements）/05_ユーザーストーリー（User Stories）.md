# ユーザーストーリー

## 概要

各ロールのユーザーストーリーを「〜として、〜したい。なぜなら〜だから。」の形式で定義する。

---

## 現場作業員（field_worker）のユーザーストーリー

| ID | ユーザーストーリー | 受入条件 | 優先度 |
|----|----------------|---------|-------|
| US-W-001 | 現場作業員として、スマートフォンで日報を入力したい。なぜなら現場から事務所に戻らずに報告できるから。 | モバイル対応UI、オフライン入力対応 | 高 |
| US-W-002 | 現場作業員として、AIに作業内容を補完してもらいたい。なぜなら日報の文章作成が苦手だから。 | キーワード入力で文章生成、編集可能 | 高 |
| US-W-003 | 現場作業員として、作業写真をその場でアップロードしたい。なぜなら後から整理する手間を省きたいから。 | スマートフォンカメラから直接アップロード | 高 |
| US-W-004 | 現場作業員として、ヒヤリハットをすぐに報告したい。なぜなら記憶が鮮明なうちに記録したいから。 | 発生場所・状況・写真を簡単に入力 | 高 |
| US-W-005 | 現場作業員として、日報の承認状況をリアルタイムで確認したい。なぜなら差し戻しがあれば早めに対応したいから。 | 承認状況のプッシュ通知 | 中 |
| US-W-006 | 現場作業員として、過去の日報を参照したい。なぜなら前日の作業内容を確認して継続作業に活かしたいから。 | 過去日報の閲覧・コピー機能 | 中 |
| US-W-007 | 現場作業員として、ナレッジベースで作業方法を検索したい。なぜなら不明な作業手順をすぐに確認したいから。 | 日本語全文検索、スマートフォン対応 | 中 |
