# セキュリティテスト計画

## 概要

本計画書は、ServiceHub Construction Platform のセキュリティテストの計画を定義する。OWASP Top 10に基づく脆弱性対策の確認、ペネトレーションテスト、コンテナセキュリティスキャンを実施する。

## ペネトレーションテストシナリオ

### PEN-001: 認証バイパステスト
```
目的: JWT検証をバイパスできないことを確認
手順:
1. 無効な署名のJWTトークンを使用してAPIアクセス試行
2. 期限切れトークンを使用してAPIアクセス試行
3. 異なるユーザーのトークンを使用した権限昇格試行
期待結果: 全試行で401/403エラーが返却される
```

### PEN-002: SQLインジェクションテスト
```
目的: SQLインジェクション脆弱性がないことを確認
手順:
1. 検索フィールドにSQLインジェクションペイロードを入力
2. ID フィールドに SQLインジェクション試行
期待結果: エラーレスポンスが返るが、DBへの影響なし
```

### PEN-003: IDOR（オブジェクト直接参照）テスト
```
目的: 他ユーザーのリソースに不正アクセスできないことを確認
手順:
1. ユーザーAとしてログイン
2. ユーザーBが所有するリソースのIDで直接アクセス
期待結果: 403 Forbidden が返却される
```

### PEN-004: ファイルアップロードセキュリティテスト
```
目的: 悪意あるファイルのアップロード・実行ができないことを確認
手順:
1. 実行可能ファイル（.exe, .sh）のアップロード試行
2. WebShellファイルのアップロード試行
3. ファイルサイズ上限を超えたアップロード試行
期待結果: 全試行で400エラーが返却される
```

## OWASP ZAP 使用方法

```bash
# OWASP ZAPを使った自動スキャン
docker run -v $(pwd):/zap/wrk/:rw \
  -t ghcr.io/zaproxy/zaproxy:stable \
  zap-baseline.py \
  -t https://test.servicehub.internal \
  -r zap_report.html \
  --hook=/zap/wrk/zap-hooks.py

# APIスキャン（OpenAPI仕様使用）
docker run -v $(pwd):/zap/wrk/:rw \
  -t ghcr.io/zaproxy/zaproxy:stable \
  zap-api-scan.py \
  -t /zap/wrk/openapi.yaml \
  -f openapi \
  -r zap_api_report.html
```

## 脆弱性の重大度分類と対応

| 重大度 | 定義 | 対応期限 | リリース可否 |
|-------|------|---------|-----------|
| CRITICAL | リモートコード実行・完全なデータ漏洩 | 即日修正 | 修正前リリース不可 |
| HIGH | 重大な認証バイパス・大規模データ漏洩 | 3日以内修正 | 修正前リリース不可 |
| MEDIUM | 限定的な情報漏洩・部分的な機能悪用 | 1週間以内修正 | 条件付きリリース可 |
| LOW | 情報開示・軽微な問題 | 次リリースまでに修正 | リリース可 |

---

## 合否基準

- CRITICAL・HIGH脆弱性が0件
- MEDIUM以下の脆弱性は全件記録・対応計画あり
- セキュリティテストレポートが作成・承認済み
- 再テストで修正確認済み
