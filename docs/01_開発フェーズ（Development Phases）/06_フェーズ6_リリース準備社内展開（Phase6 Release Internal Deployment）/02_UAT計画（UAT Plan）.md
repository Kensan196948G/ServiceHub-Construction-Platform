# UAT（ユーザー受入テスト）計画

## 概要

ユーザー受入テスト（UAT）は、実際のエンドユーザーがシステムを使用し、業務要件を満たしているかを確認するテストである。本計画書は、UATの実施手順・参加者・シナリオ・合否基準を定義する。

## UATフィードバック収集方法

### アンケート形式

```
1. 画面の操作性について（5段階評価）
   □ 非常に使いやすい　□ 使いやすい　□ 普通　□ 使いにくい　□ 非常に使いにくい

2. 業務要件を満たしているか（5段階評価）
   □ 完全に満たしている　□ ほぼ満たしている　□ 普通　□ やや不満　□ 満たしていない

3. 改善してほしい点（自由記述）:
   ___________________________________________

4. 特に良かった機能（自由記述）:
   ___________________________________________

5. 本番で使用したいと思うか（5段階評価）
   □ 非常にそう思う　□ そう思う　□ 普通　□ あまり思わない　□ 思わない
```

## 合否基準

| 指標 | 合格基準 |
|-----|---------|
| シナリオ完了率 | ≥95%のシナリオが正常完了 |
| ユーザー満足度 | 平均スコア ≥4.0/5.0 |
| 業務要件充足率 | ≥90%のユーザーが「満たしている」と評価 |
| 本番利用意向 | ≥80%のユーザーが「使用したい」と評価 |
| P1・P2指摘 | UAT中に発見されたP1・P2問題が全件修正済み |

---

## 問題発見時の対応手順

1. **問題報告**：参加者がUATフィードバックシートに記録
2. **即時確認**：ファシリテーターが問題を確認・記録
3. **優先度判定**：開発チームが重大度を判定（P1〜P4）
4. **対応**：P1は即日修正、P2は翌日修正、P3〜4はリリース前修正
5. **確認**：修正後に参加者が再確認
