# 運用移行計画

## 概要

本計画書は、既存業務からServiceHub Construction Platform への移行計画を定義する。データ移行、既存システムの切り替え、ユーザー研修、運用チームへの引き継ぎを計画的に実施する。

---

## 移行対象システム・業務

| 既存システム/業務 | 移行先機能 | 移行難易度 |
|---------------|---------|---------|
| Excelによる案件管理 | 工事案件管理モジュール | 高 |
| 紙の日報・手書き | 日報管理モジュール | 中 |
| 共有フォルダの写真管理 | 写真・資料管理モジュール | 中 |
| 安全点検チェックシート（紙） | 安全品質管理モジュール | 低 |
| Excelによる原価管理 | 原価・工数管理モジュール | 高 |

---

## データ移行計画

### 移行対象データ

| データ種別 | 移行方法 | 移行量（概算） | 担当 |
|---------|---------|-------------|------|
| 既存案件マスタ | ETLスクリプト | 約200件 | 開発チーム |
| ユーザーマスタ | CSVインポート | 約150名 | IT部門 |
| 費目マスタ | CSVインポート | 約50件 | 経理部門 |
| 過去日報（直近1年分） | ETLスクリプト | 約5,000件 | 開発チーム |
| 過去写真（直近6ヶ月） | バッチ転送 | 約10,000枚 | 開発チーム |

### データ移行手順

```bash
# 1. 移行前バリデーション
python migrate/validate_source.py --source=excel --output=validation_report.csv

# 2. ユーザーマスタ移行
python migrate/import_users.py --file=users_master.csv --dry-run
python migrate/import_users.py --file=users_master.csv --execute

# 3. 案件マスタ移行
python migrate/import_projects.py \
  --source=projects_excel.xlsx \
  --mapping=migration_mapping.yaml \
  --dry-run
python migrate/import_projects.py \
  --source=projects_excel.xlsx \
  --mapping=migration_mapping.yaml \
  --execute

# 4. 移行後整合性チェック
python migrate/validate_migration.py --check-all
```

## 移行リスク管理

| リスク | 対策 |
|-------|------|
| データ移行エラー | 移行前バリデーション・移行後整合性チェックの実施 |
| ユーザーの操作習熟不足 | 並行稼働期間・研修・マニュアル整備 |
| 既存データの形式不整合 | ETLマッピング設計・手動補正対応 |
| 移行作業の遅延 | 移行スケジュールにバッファを設ける |
