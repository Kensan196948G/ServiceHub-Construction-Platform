# Operation Loops

曜日別運用資料の強みを残しつつ、Claude Code で継続しやすい形に整理した運用ループです。

## 推奨ループ

`Monitor -> Build -> Verify -> Improve`

## 1. Monitor

### やること

- 要件、設計、README の差分確認
- Git 状態確認
- 失敗中テストや CI 状況確認
- 今日やる 1〜3 タスクへの分解

### 出力

- 目的
- 成功条件
- 今回触るファイルや領域

## 2. Build

### やること

- まず設計メモを 3〜5 行で固める
- 変更を 1 テーマに絞る
- 関連テストを追加する

### 禁止

- ついでの大規模整理
- 成功条件が曖昧なままの横展開

## 3. Verify

### やること

- 変更近傍テスト
- lint / typecheck
- build または起動確認
- セキュリティや認可への影響確認

### 出力

- 実行コマンド
- 成否
- 未確認項目

## 4. Improve

### やること

- 命名整理
- コメント最適化
- README / docs 更新
- 再開メモ作成

## 5. 実運用のコツ

- 厳密な「1時間後に必ず Monitor」より、フェーズ完了時の切替を優先
- 小変更なら `Monitor -> Build -> Verify` だけでもよい
- 大変更だけ `Improve` と複数エージェントを厚く使う

## 6. 8時間セッションの扱い

8時間は「強制的に前進を止める境界」として使う。

終了時は最低でも次を残す。

- 変更サマリ
- テスト結果
- 残課題
- 再開ポイント
