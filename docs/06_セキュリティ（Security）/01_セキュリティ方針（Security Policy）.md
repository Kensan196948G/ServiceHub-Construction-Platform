# セキュリティ方針

## 概要

ServiceHub Construction Platform のセキュリティ方針は、ISO27001（情報セキュリティマネジメントシステム）に準拠し、建設業の業務システムとして求められる高いセキュリティレベルを実現する。

---

## セキュリティ目標

| 目標 | 内容 |
|-----|------|
| 機密性（Confidentiality） | 権限のないユーザーへの情報漏洩防止 |
| 完全性（Integrity） | データの不正な変更・破壊の防止 |
| 可用性（Availability） | 正当なユーザーが必要な時にサービスを利用できる保証 |

---

## 準拠規格・フレームワーク

| 規格/フレームワーク | 適用範囲 |
|----------------|---------|
| ISO/IEC 27001:2022 | 情報セキュリティマネジメント全般 |
| NIST CSF2.0 | サイバーセキュリティフレームワーク |
| OWASP Top 10 | Webアプリケーションセキュリティ |
| CIS Controls v8 | セキュリティ管理策 |

---

## セキュリティ管理策（ISO27001対応）

### A.5 情報セキュリティポリシー
- セキュリティ方針の文書化・周知・定期レビュー
- 年次セキュリティレビューの実施

### A.6 情報セキュリティの組織
- セキュリティ責任者の任命
- 開発チームへのセキュリティ教育

### A.8 資産管理
- システム資産台帳の管理
- データ分類（機密・社内限・公開）

### A.9 アクセス制御
- RBAC（ロールベースアクセス制御）の実装
- 最小権限の原則の適用
- 定期的なアクセス権レビュー（四半期）

### A.10 暗号化
- データ転送時のTLS 1.2以上
- パスワードのbcryptハッシュ化
- センシティブデータの保存時暗号化

### A.12 運用セキュリティ
- マルウェア対策
- バックアップ管理
- ログ管理（5年間保管）

### A.14 システム取得・開発・保守
- セキュアコーディング規約の遵守
- コードレビューでのセキュリティチェック
- 依存パッケージの脆弱性管理

### A.16 情報セキュリティインシデント管理
- セキュリティインシデント対応手順
- インシデントの記録・分析

---

## NIST CSF2.0 対応

| 機能 | 実装内容 |
|-----|---------|
| Govern（ガバナンス） | セキュリティ方針・リスク管理プロセス |
| Identify（特定） | 資産管理・リスク評価 |
| Protect（保護） | RBAC・暗号化・セキュリティ教育 |
| Detect（検知） | ログ監視・SIEM・異常検知 |
| Respond（対応） | インシデント対応手順・ITSM連携 |
| Recover（回復） | バックアップ・DR訓練 |
