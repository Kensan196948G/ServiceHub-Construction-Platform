# UI/UX設計

## デザイン方針

| 方針 | 内容 |
|-----|------|
| モバイルファースト | スマートフォンでの現場入力を最優先で設計 |
| シンプル・直感的 | 建設現場での利用を想定した大きなボタン・シンプルなフォーム |
| レスポンシブ | PC・タブレット・スマートフォン全対応 |
| アクセシビリティ | WCAG 2.1 AA準拠 |
| 高速 | 初回ロード3秒以内、操作レスポンス1秒以内 |
| ダークモード | 屋外利用を考慮したダークモード対応 |

## 画面構成

### ナビゲーション構造

```
/ (ダッシュボード)
├── /projects (案件一覧)
│   ├── /projects/new (新規案件作成)
│   └── /projects/{id} (案件詳細)
│       ├── /projects/{id}/members (メンバー管理)
│       └── /projects/{id}/progress (進捗管理)
├── /daily-reports (日報一覧)
│   ├── /daily-reports/new (日報作成)
│   └── /daily-reports/{id} (日報詳細)
├── /photos (写真管理)
├── /safety (安全品質管理)
│   ├── /safety/checks (安全点検)
│   ├── /safety/hazards (ヒヤリハット)
│   └── /safety/quality (品質検査)
├── /costs (原価管理)
│   ├── /costs/budgets (予算管理)
│   └── /costs/actuals (実績原価)
├── /itsm (ITSM管理)
│   ├── /itsm/incidents (インシデント)
│   ├── /itsm/problems (問題管理)
│   └── /itsm/changes (変更管理)
├── /knowledge (ナレッジ)
└── /ai-chat (AIチャット)
```

## モバイル対応

### スマートフォン向けの特別対応

| 機能 | PC | モバイル |
|-----|----|----|
| 日報入力 | 全項目フォーム | AI補完を前面に出したシンプルUI |
| 写真アップロード | ドロップゾーン | カメラ直接起動ボタン |
| 安全点検 | 表形式 | スクロール式チェックリスト |
| ナビゲーション | サイドバー | ボトムナビゲーション |
| 通知 | Webプッシュ | プッシュ通知（PWA） |
