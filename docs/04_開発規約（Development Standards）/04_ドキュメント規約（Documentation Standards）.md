# ドキュメント規約

本プロジェクトのドキュメントはMarkdown形式で作成し、GitHubリポジトリ内で管理する。ドキュメントもコードと同様にバージョン管理し、常に最新の状態を維持する。

## Markdown記述規則

### 見出し規則

\`\`\`markdown
# ドキュメントタイトル（H1は1つのみ）

## 大セクション（H2）

### 小セクション（H3）

#### 詳細（H4、多用しない）
\`\`\`

### テーブル規則

- ヘッダー行には必ず `|---|---|` の区切り行を入れる
- テーブルの先頭列は意味のある項目名を使う

### コードブロック規則

- 言語を必ず指定する（\`\`\`python, \`\`\`yaml, \`\`\`sql, \`\`\`bash 等）
- 長いコードは必要な部分のみ抜粋し、省略箇所は `# ...` で示す

### Mermaidダイアグラム規則

- フローチャート: `graph LR` or `graph TD`
- シーケンス図: `sequenceDiagram`
- ガントチャート: `gantt`
- 状態遷移図: `stateDiagram-v2`
- ER図: `erDiagram`

---

## ドキュメント更新ルール

1. **コードと同時更新**：コード変更に伴うドキュメント更新はコードのPRと同じPRで行う
2. **レビュー必須**：ドキュメントの変更もコードレビューと同様のレビューを実施
3. **バージョン管理**：ドキュメントの大幅変更時はファイル末尾に変更履歴を記載
4. **日本語統一**：全ドキュメントは日本語で記述（コードのみ英語可）
5. **リンクの維持**：他ドキュメントへのリンクが壊れていないことを確認
