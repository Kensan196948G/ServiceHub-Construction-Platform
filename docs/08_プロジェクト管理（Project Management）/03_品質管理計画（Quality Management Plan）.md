# 🏆 品質管理計画（Quality Management Plan）

## 品質方針
「動くソフトウェアを小さく・確実に・安全に届ける」

## 品質目標

| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| テストカバレッジ | 80%以上 | pytest-cov / Istanbul |
| バグ密度 | 5件/KLOC以下 | バグ管理DB |
| 重大障害（Critical） | 0件 | インシデント管理 |
| セキュリティ脆弱性（Critical/High） | 0件 | 脆弱性スキャン |
| APIレスポンス（P95） | 500ms以下 | Prometheus |
| コードレビュー実施率 | 100% | GitHub PR |

## 品質ゲート（フェーズ完了条件）

各フェーズ終了時に以下の品質ゲートを通過すること：

```
✅ 単体テスト全件PASS
✅ コードレビュー完了
✅ lintエラー 0件
✅ セキュリティスキャン PASS
✅ docs更新完了
✅ CIパイプライン GREEN
```

## レビュープロセス
1. **セルフレビュー**: 実装者自身でチェックリスト確認
2. **コードレビュー**: 1名以上のレビュアーによるPRレビュー
3. **QAレビュー**: テスト結果確認
4. **セキュリティレビュー**: セキュリティ観点の確認（フェーズ単位）

## 品質改善プロセス
- バグ原因分析（RCA）→ 再発防止策
- 週次メトリクス確認
- 月次品質レポート作成
