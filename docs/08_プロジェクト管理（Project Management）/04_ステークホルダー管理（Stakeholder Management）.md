# 👥 ステークホルダー管理（Stakeholder Management）

## ステークホルダー一覧

| 名前/役割 | 組織 | 関与度 | 主な関心事 |
|----------|------|--------|-----------|
| プロジェクトスポンサー | 経営層 | 高 | ROI・スケジュール |
| IT部門責任者 | IT部門 | 高 | セキュリティ・ITSM |
| 現場監督 | 建設現場 | 高 | 使いやすさ・スマホ対応 |
| 原価担当者 | 経理部門 | 中 | 原価精度・レポート |
| 開発チーム | IT部門 | 高 | 技術・品質 |
| 運用チーム | IT部門 | 高 | 安定性・運用性 |
| 監査担当者 | 内部監査 | 中 | 監査証跡・コンプライアンス |

## コミュニケーション計画

| 対象 | 方法 | 頻度 | 内容 |
|------|------|------|------|
| 経営層 | 月次報告書 | 月1回 | 進捗・リスク・KPI |
| IT責任者 | 週次会議 | 週1回 | 技術進捗・Issue |
| 現場ユーザー | UAT参加 | フェーズ6 | デモ・フィードバック |
| 開発チーム | デイリー | 毎日 | タスク・ブロッカー |

## 承認フロー

```
要件変更 → PM確認 → IT責任者承認 → スポンサー承認（重大変更時）
```
