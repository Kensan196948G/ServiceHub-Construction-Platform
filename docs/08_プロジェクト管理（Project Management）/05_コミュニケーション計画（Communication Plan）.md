# 📢 コミュニケーション計画（Communication Plan）

## ツール一覧

| ツール | 用途 |
|--------|------|
| GitHub Issues | タスク管理・バグ管理 |
| GitHub Projects | 進捗ボード |
| GitHub PR | コードレビュー |
| Slack / Teams | 日常コミュニケーション |
| 週次会議 | 進捗確認・意思決定 |

## 定例会議

| 会議名 | 頻度 | 参加者 | 内容 |
|--------|------|--------|------|
| デイリースタンドアップ | 毎日 | 開発チーム | 進捗・ブロッカー確認 |
| 週次進捗会議 | 週1回 | 開発＋IT責任者 | フェーズ進捗・リスク |
| フェーズレビュー | フェーズ末 | 全関係者 | 成果物レビュー・次フェーズ確認 |
| 月次報告 | 月1回 | 経営層＋PM | KPI・進捗・課題報告 |

## エスカレーションパス
```
担当者 → Tech Lead → PM → IT責任者 → スポンサー
（緊急時はスキップ可）
```

## 開発ループ コミュニケーション出力
各ループ終了時に以下を出力・共有：
```yaml
summary:
  phase: Monitor/Development/Verify/Improvement
  actions: 実施内容
  results: 結果
  issues: 課題
  next_steps: 次アクション
```
